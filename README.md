# whocalleditright.com

A public Brier scorecard auditing the most-cited hedge fund managers and pundits. Auto-updated from SEC EDGAR 13F filings + verifiable public statements.

**Live**: https://whocalleditright.com (once deployed)
**Data repo**: https://github.com/alex-jb/whocalleditright-data

---

## Look + feel

Editorial + brutalist. Off-white `#F5F4EE` background, Inter Tight Black display headlines, JetBrains Mono for tickers, single muted-orange `#E45A2A` accent. Each fund manager is rendered as a 1970s-Time-Magazine-style stylized portrait (not photo, not the subject's likeness). No SaaS gradients, no neon, no card-lift hover effects.

## Why this exists

Stanley Druckenmiller dumped GOOGL and AMZN in his Q1 2026 13F filing (posted 2026-05-15). Three weeks later, Friday 2026-06-05 hit and the NASDAQ dropped 4.2%. He looks prescient. But nobody tracks this systematically.

So I did. 19 managers, scored by Brier, public.

## Stack

- **Next.js 16** + React 19 server components
- **TypeScript** strict
- **Tailwind CSS 4** with custom editorial palette
- **Inter Tight** + **JetBrains Mono** via `next/font/google`
- **ISR** every 30 minutes — zero client-side data fetching
- **Data source**: GitHub raw (`alex-jb/whocalleditright-data`), no DB, no auth
- **Portraits**: OpenAI `gpt-image-1`, editorial 1970s Time Magazine style — script in `scripts/gen_portraits.sh`
- **Deployed**: Vercel

## Local dev

```bash
npm install
npm run dev
# → http://localhost:3000
```

By default `NEXT_PUBLIC_WHOSECALL_DATA_MODE=local` reads the seed JSON from `./data/funds/*.json`. To pull live data from GitHub raw:

```bash
NEXT_PUBLIC_WHOSECALL_DATA_MODE=remote npm run dev
```

## AI portraits

19 stylized portraits live in `public/portraits/<slug>.png`. The repo ships **procedural placeholder PNGs** generated by `scripts/make_placeholders.mjs` so the UI renders cleanly out of the box.

To replace them with real OpenAI `gpt-image-1` outputs:

```bash
# 1. Add your key
echo 'OPENAI_API_KEY=sk-...' >> .env.local

# 2. Run the generator (all 19, ~5 min, ~$0.76 at 1024x1024)
bash scripts/gen_portraits.sh

# Or just one:
bash scripts/gen_portraits.sh druckenmiller
```

The style prompt lives in `scripts/gen_portraits.mjs`. Each portrait is a metaphor (e.g. Druckenmiller stands in front of a wall of stacked old monitors), explicitly NOT a description of the real person — to avoid right-of-publicity issues.

## How data syncs from the local 13F tracker

This is the public-facing surface. The data pipeline lives in `~/.solo-founder-os/markets/` on Alex's laptop:

1. **`13f_tracker.py`** polls SEC EDGAR daily for new 13F-HR filings from the 19 tracked CIKs.
2. New deltas → `~/.solo-founder-os/state/13f/<slug>.jsonl`.
3. **`whocalleditright_sync.py`** (to be written) reads the JSONL, computes Brier via the same logic as `lib/brier/compute.ts`, and emits one `<slug>.json` per fund matching the schema in `lib/types.ts`.
4. The script commits + pushes to `github.com/alex-jb/whocalleditright-data` (separate repo, public, MIT).
5. This Next.js app re-fetches every 30 min via ISR.

```
SEC EDGAR
    ↓ daily cron (13f_tracker.py)
~/.solo-founder-os/state/13f/*.jsonl
    ↓ whocalleditright_sync.py
github.com/alex-jb/whocalleditright-data/data/funds/*.json
    ↓ ISR fetch (revalidate=1800)
whocalleditright.com
```

This mirrors the `predictions-feed` pattern Alex already uses for the contrarian signals JSON. Same shape: local script writes JSON → push to public data repo → Next.js site renders.

## Brier scoring (v0)

See `lib/brier/compute.ts`. Simplified:

- NEW or INCREASED → BULLISH
- EXITED or DECREASED → BEARISH
- Bullish correct = price up; bearish correct = price down
- Brier = `(forecast - outcome)²` scaled by magnitude clip `|move%| / 20`
- Lower = better. 0 = perfect, 1 = max wrong.

Limits in [methodology](https://whocalleditright.com/methodology).
